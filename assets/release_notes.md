# Dokkaebi 릴리즈 노트

## v0.4.0 (2026-04-23)

### 새로운 기능
- **워크스페이스 그룹 색상 지정**: 워크스페이스 그룹을 우클릭해 6색 팔레트에서 아이콘 색을 선택할 수 있고, 같은 색을 다시 클릭하면 기본 색상으로 돌아갑니다. 선택한 색상은 앱 재시작 후에도 유지
- **Claude Code 서브에이전트 뷰 탭**: Claude Code 가 Task 도구로 서브에이전트를 실행하면 활성 pane 을 오른쪽/아래쪽으로 split 해 진행 상황 뷰 탭을 자동으로 엽니다. 도구 호출 로그와 최종 응답을 실시간 확인 가능
- **서브에이전트 패널 위치 설정**: 설정 > 알림 > "패널" 섹션의 "서브에이전트 패널 위치" 로 오른쪽(기본)·하단 split 중 선택
- **서브에이전트 뷰 자동 생성 토글**: 설정 > 알림 > "패널" 섹션의 "서브에이전트 뷰 탭" 토글로 자동 생성 제어. 끈 상태에서도 이미 열린 탭은 유지
- **Claude Code 훅 파서(jq) 번들 포함**: Dokkaebi 인스톨러가 `jq.exe`(MIT, jqlang v1.8.1)를 함께 설치해 사용자가 별도 설치 없이 서브에이전트 로그·토스트 본문이 정상 표시됨

### UI/UX 개선
- **워크스페이스 목록 항목 아이콘**: 좌측 워크스페이스 그룹 패널 각 항목 이름 앞에 터미널 아이콘이 추가되어 워크스페이스가 터미널 묶음임을 한눈에 식별
- **릴리즈 노트 탭 한글화**: 한글 UI에서 릴리즈 노트 탭 이름이 "릴리즈 노트" 로 표시됨. 일반 Markdown 미리보기 탭도 한글에서는 파일 이름만 노출
- **서브에이전트 뷰 세로 스크롤**: 서브에이전트 뷰 탭에 세로 스크롤바가 추가되어 로그가 길어져도 전체 내용 탐색 가능
- **서브에이전트 탭 묶음 배치**: 여러 서브에이전트를 병렬 실행해도 동일한 split pane 안에 탭으로 모여 표시
- **환영 탭 도움말 확장**: 도움말 섹션에 서브에이전트 뷰 탭 활성화 안내가 추가되어 신규 기능 발견성이 향상되고, Claude Code 알림 설정 경로 안내도 실제 설정 메뉴 구조에 맞게 정정됨
- **서브에이전트 뷰 스트리밍 중 스크롤 유지**: 로그가 새로 추가돼도 사용자가 보고 있던 위치와 드래그 선택이 리셋되지 않고 꼬리에만 증분 반영
- **서브에이전트 뷰 로그 정확도**: 동일 세션에서 다른 서브에이전트나 부모 에이전트의 메시지가 뷰에 섞여 나오던 경로를 차단해 자기 로그만 표시

### 정리
- **Claude Code 훅 처리 지연 단축**: 서브에이전트 훅에서 호출하던 외부 파서(jq) 프로세스 수를 훅당 5회에서 1회로 통합해 훅 응답 속도 개선
- **transcript 자동 정리 시작 지연 완화**: 앱 시작 시 24시간 이내 이미 정리한 경우 재스캔을 건너뛰어 부팅 I/O 중복 제거

## v0.3.0 (2026-04-21)

### 새로운 기능
- **Windows 자동 실행 설정**: 설정 > 일반의 "자동 실행" 토글로 Windows 로그인 시 Dokkaebi 자동 실행을 켜고 끌 수 있음. 활성화하면 다음 로그인부터 자동으로 실행되고, 비활성화하거나 앱을 제거하면 자동 실행 등록이 해제됨
- **Claude Code 작업 알림**: Claude Code의 작업 완료·입력 대기·권한 요청을 Dokkaebi 토스트 알림으로 표시. 토스트 본문에 "그룹 / 탭" 라인이 함께 노출되어 발신 위치 식별 가능
- **터미널 탭/그룹 인디케이터**: 알림 도착 터미널 탭에는 dot, 해당 터미널이 속한 비활성 워크스페이스 그룹에는 배지 표시
- **알림 플러그인 관리**: 설정 > 알림 > Claude Code의 [설치]/[제거] 버튼으로 활성화 제어. 인스톨러에 번들되어 별도 다운로드 불필요

### UI/UX 개선
- **알림 메뉴 재구성**: "Claude Code 작업 완료 알림음" 항목이 "Claude Code 플러그인" + "작업 알림" 두 항목으로 재구성되어 플러그인 설치 상태가 화면에 즉시 표시됨
- **토스트 자동 사라짐 시간 조정**: 작업 완료·입력 대기 토스트는 기본 5초 후 자동으로 사라지고, 설정 > 알림 > "토스트 팝업" 섹션에서 5~300초 범위 조정 가능. 권한 요청 토스트는 사용자가 닫을 때까지 유지
- **토스트 중첩 방지**: 입력 대기 토스트가 떠 있을 때 새 작업 완료 알림이 도착하면 이전 토스트는 자동 정리. 여러 창에 같은 이름 탭이 있어도 발신 터미널이 속한 창에만 토스트 표시
- **Claude Code 토스트 내용 풍부화**: 작업 완료 시 직전 프롬프트와 응답 요약을 `"프롬프트 → 응답"` 형식으로, 권한 요청 시 도구 이름과 실행할 명령·파일 preview를, 입력 대기 시 Claude가 띄운 실제 질문을 토스트에서 바로 확인 가능
- **토스트 다국어 적용**: 토스트 제목·베이스 문구가 설정된 UI 언어(한국어/English)에 맞춰 즉시 전환
- **CLI 바이너리 아이콘 적용**: 설치 폴더와 탐색기에서 `dokkaebi-cli.exe`도 메인 앱과 동일한 도깨비 아이콘으로 표시되어 비주얼 일관성 향상
- **워크스페이스 그룹 패널 항목 확대**: 좌측 워크스페이스 그룹 패널의 항목 세로 높이를 약 30% 크게 조정해 클릭 영역과 가독성 개선
- **메모장 기본 복원 켜짐**: 설정 > 패널 > 메모장 복원 기본값이 켜짐으로 바뀌어 작성 중이던 메모가 앱 재시작 후에도 자동으로 유지됨

### 보안
- **rustls-webpki 보안 패치**: Dependabot 보안 경고 2건 해결을 위해 `rustls-webpki`를 0.103.11 → 0.103.12로 업데이트 (TLS 인증서 검증 관련 low 등급 경고 2건)

### 정리
- **마커 파일 기반 알림 시스템 폐기**: 기존 "Claude Code 작업 완료 알림음"이 사용하던 임시 파일 폴링 방식을 제거하고 IPC 기반 알림으로 전환. 기존 사용자 환경의 잔여 hook은 부팅 시 1회 자동으로 정리됨
- **언인스톨 시 Claude Code 훅 자동 정리**: 인스톨러로 Dokkaebi를 제거하면 `~/.claude/settings.json`에 등록된 Dokkaebi 플러그인 항목이 함께 제거됨. 사용자가 편집한 다른 Claude Code 설정은 보존되고, 자동 업데이트 재설치 경로에서는 hook을 유지해 업데이트 후 재등록 불필요

### 버그 수정
- **Copilot Chat reasoning effort 전송 오류 수정**: Copilot proxy가 검증을 강화하면서 reasoning effort가 지정되지 않은 상태로 메시지를 보내면 거부되던 문제 수정 (상류 Zed v0.232.3 반영)
     ([자세한 내용은 공식 Zed 릴리즈 노트 참고](https://github.com/zed-industries/zed/releases/tag/v0.232.3))

## v0.2.0 (2026-04-18)

### 새로운 기능
- **업스트림 Zed v0.232.2 백포트**: ESLint 3.0.24, LanguageAwareStyling, ACP slash commands 복원, Markdown HTML 정렬, 프로젝트 패널 정렬 순서, SSH nickname 표시, 이미지 멘션 이름, OpenAI reasoning_effort, JSX 컴포넌트 하이라이팅, 카드 레이아웃 패딩, 병합 충돌 상태바 표시기, SVG 폰트 fallback 등 20+ PR 이식
  ([자세한 내용은 공식 Zed 릴리즈 노트 참고](https://github.com/zed-industries/zed/releases/tag/v0.232.2))
- **릴리즈 노트 메뉴**: 설정 메뉴에서 릴리즈 노트를 직접 확인 가능
- **메모장 패널 단축키**: `Ctrl+Shift+M`으로 메모장 패널 토글
- **UI 언어 변경 UI**: 설정 > 일반에서 언어(시스템 언어 / English / 한국어) 선택 가능. 시스템 언어가 기본값이며 OS 언어를 자동 감지하고, 변경 시 재시작 없이 즉시 적용
- **자동 업데이트 후 릴리즈 노트 표시**: 앱 내부 자동 업데이트로 새 버전이 설치되면 설치가 완료되는 즉시 앱이 자동 재실행되고 릴리즈 노트가 1회 자동으로 열림. 설정 > 일반 > 자동 업데이트의 "릴리즈 노트" 토글로 켜고 끌 수 있음

### UI/UX 개선
- **대규모 i18n 한글화**: 다이얼로그, 툴팁, 버튼, 라벨, 피커 placeholder, 드롭다운, 확장 카드, 키맵 편집기, 상태바 LSP 버튼, ETW 트레이싱 알림 등 100+ 문자열 한글 적용
- **워크스페이스 그룹 패널**: 항목 간 4px 간격 추가로 가독성 향상
- **프로젝트 패널 정렬**: 설정 UI에서 정렬 모드/순서를 직관적인 위치로 이동
- **Windows 앱 목록 표시명 단순화**: "Dokkaebi 0.2.0" → "Dokkaebi"로 표시. 버전은 별도 줄(`DisplayVersion`)에 그대로 노출
- **정보(About) 팝업 리뉴얼**: 상류 Zed 스타일의 독립 윈도우로 전환하고 상단에 앱 아이콘을 추가. 타이틀·버전·크레딧·확인 버튼은 기존 내용 유지. 듀얼 모니터 환경에서는 현재 Dokkaebi 메인 앱이 있는 모니터 중앙에 표시

### 보안
- **wasmtime 33 → 36 업그레이드**: Dependabot 보안 경고 해결 (critical 2, moderate 10, low 3)
- **설치 다운그레이드 보호**: 이미 설치된 버전이 더 높으면 설치 시 경고 후 진행 여부 확인. 앱 자동 업데이트 경로(silent)에서는 다운그레이드를 무조건 차단하여 의도치 않은 버전 되돌림 방지

### 정리
- **REPL/Notebook 크레이트 완전 제거**: Windows 환경에서 사용 불가한 Jupyter 관련 코드 정리
- **설정 메뉴 정리**: 불필요한 키맵 파일 메뉴 항목 제거

### 버그 수정
- Non-ASCII 아카이브 스레드 제목 크래시 수정 (한국어 관련)
- 프로젝트 심볼 UTF-8 패닉 수정
- 설정 입력 필드 blur 시 데이터 손실 수정
- Hover popover delay 설정 경로 수정
- 프로필 선택기 Shift+Tab 재렌더링 수정
- Rules library 호버 시 활성 규칙 유지
