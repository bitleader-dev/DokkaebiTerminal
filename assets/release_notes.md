# Dokkaebi 릴리즈 노트

## v0.4.0 (2026-04-23)

### 새로운 기능
- **Claude Opus 4.7 모델 지원**: 언어 모델 선택에 Claude Opus 4.7 이 추가되어 Anthropic·Bedrock·OpenCode 제공자 모두에서 선택 가능. 1M 토큰 컨텍스트와 128K 출력 토큰을 지원
- **워크스페이스 그룹 색상 지정**: 워크스페이스 그룹을 우클릭해 6색 팔레트에서 아이콘 색을 선택할 수 있고, 같은 색을 다시 클릭하면 기본 색상으로 돌아갑니다. 선택한 색상은 앱 재시작 후에도 유지
- **Claude Code 서브에이전트 뷰 탭**: Claude Code 가 Task 도구로 서브에이전트를 실행하면 활성 pane 을 오른쪽/아래쪽으로 split 해 진행 상황 뷰 탭을 자동으로 엽니다. 도구 호출 로그와 최종 응답을 실시간 확인 가능
- **서브에이전트 패널 위치 설정**: 설정 > 알림 > "패널" 섹션의 "서브에이전트 패널 위치" 로 오른쪽(기본)·하단 split 중 선택
- **서브에이전트 뷰 자동 생성 토글**: 설정 > 알림 > "패널" 섹션의 "서브에이전트 뷰 탭" 토글로 자동 생성 제어. 끈 상태에서도 이미 열린 탭은 유지
- **Claude Code 훅 파서(jq) 번들 포함**: Dokkaebi 인스톨러가 `jq.exe`(MIT, jqlang v1.8.1)를 함께 설치해 사용자가 별도 설치 없이 서브에이전트 로그·토스트 본문이 정상 표시됨
- **즐겨찾기 모델에 thinking/effort/fast 설정 저장**: 모델을 즐겨찾기로 등록하면 현재 thinking·effort·fast mode 설정이 함께 기록되고, 해당 모델로 돌아오면 이전 설정이 자동 복원됨
- **모델 설정 토글이 즐겨찾기와 동기화**: thinking / effort 단계 / fast mode 를 UI 에서 바꾸면 기본 모델뿐 아니라 해당 즐겨찾기 항목에도 즉시 반영되어 별도 재설정 불필요
- **앱 재시작 후 fast mode 유지**: 이전에 켜둔 fast mode 가 새 스레드에서도 그대로 유지됨
- **Markdown 프리뷰 헤딩 앵커 링크**: 헤딩에 마우스를 올리면 # 아이콘으로 앵커 URL 을 복사할 수 있고, `[링크](#heading-slug)` 클릭 시 해당 헤딩으로 스크롤
- **Markdown 프리뷰 각주 지원**: `[^1]` 각주 참조를 클릭하면 해당 각주 정의로 이동하고, 정의 안의 되돌아가기 링크로 원래 위치로 복귀

### UI/UX 개선
- **에이전트 워크트리 선택기 통합**: 에이전트 패널 툴바의 "현재 워크트리 / 새 워크트리" 2지선다 드롭다운이 통합 워크트리 피커로 교체됨. 현재 브랜치·기본 브랜치·이름 지정 worktree 를 하나의 검색 가능한 팝오버에서 고를 수 있고, 툴바 라벨이 현재 활성 worktree 이름을 바로 보여줌
- **비포커스 창 프레임율 제한으로 에너지 절약**: 다른 창을 보고 있는 동안 Dokkaebi 창의 프레임 갱신을 약 30fps로 제한해 GPU·배터리 소모 감소
- **설정 검색 결과 이동 개선**: 검색 시 가장 많이 일치하는 네비게이션 항목을 자동 선택하고 해당 섹션 헤더로 스크롤해 원하는 설정을 바로 확인 가능
- **환영 탭 최근 프로젝트 이름 풍부화**: 단일 폴더뿐 아니라 여러 루트를 묶은 워크스페이스도 모든 폴더 이름을 표시하고, 원격 프로젝트도 고정 문구 대신 실제 폴더명을 노출
- **삭제된 폴더 최근 프로젝트 1주 유예**: 최근 프로젝트 목록에서 폴더가 사라져도 7일간 항목을 유지해 임시 언마운트된 드라이브·USB를 곧바로 지우지 않음
- **워크스페이스 목록 항목 아이콘**: 좌측 워크스페이스 그룹 패널 각 항목 이름 앞에 터미널 아이콘이 추가되어 워크스페이스가 터미널 묶음임을 한눈에 식별
- **릴리즈 노트 탭 한글화**: 한글 UI에서 릴리즈 노트 탭 이름이 "릴리즈 노트" 로 표시됨. 일반 Markdown 미리보기 탭도 한글에서는 파일 이름만 노출
- **서브에이전트 뷰 세로 스크롤**: 서브에이전트 뷰 탭에 세로 스크롤바가 추가되어 로그가 길어져도 전체 내용 탐색 가능
- **서브에이전트 탭 묶음 배치**: 여러 서브에이전트를 병렬 실행해도 동일한 split pane 안에 탭으로 모여 표시
- **환영 탭 도움말 확장**: 도움말 섹션에 서브에이전트 뷰 탭 활성화 안내가 추가되어 신규 기능 발견성이 향상되고, Claude Code 알림 설정 경로 안내도 실제 설정 메뉴 구조에 맞게 정정됨
- **서브에이전트 뷰 스트리밍 중 스크롤 유지**: 로그가 새로 추가돼도 사용자가 보고 있던 위치와 드래그 선택이 리셋되지 않고 꼬리에만 증분 반영
- **서브에이전트 뷰 로그 정확도**: 동일 세션에서 다른 서브에이전트나 부모 에이전트의 메시지가 뷰에 섞여 나오던 경로를 차단해 자기 로그만 표시

### 보안
- **공급망 취약점 전량 해결**: openssl(HIGH 4 + LOW 1) · rand(LOW 1) · rustls-webpki(LOW 2) 총 8건의 Dependabot 알림을 lockfile 갱신 및 AWS SDK feature 조정으로 해결하고 내부 TLS 스택을 rustls 0.23 단일 경로로 정리

### 정리
- **Claude Code 훅 처리 지연 단축**: 서브에이전트 훅에서 호출하던 외부 파서(jq) 프로세스 수를 훅당 5회에서 1회로 통합해 훅 응답 속도 개선
- **transcript 자동 정리 시작 지연 완화**: 앱 시작 시 24시간 이내 이미 정리한 경우 재스캔을 건너뛰어 부팅 I/O 중복 제거

### 버그 수정
- **구형·가상 GPU 기동 크래시 방지**: Bgra8Unorm 텍스처 포맷을 지원하지 않는 GPU 에서 Dokkaebi 가 panic 없이 Rgba8Unorm 으로 자동 전환되어 정상 실행됨 (일부 가상화·원격 데스크톱·구형 드라이버 환경)
- **AI 편집 커밋 레이스 수정**: 에이전트 편집을 git 으로 커밋하는 동시에 사용자가 다른 곳을 편집해도 에이전트 편집이 정상적으로 "수락됨" 으로 처리되고 리뷰 대기 목록에서 사라짐
- **tsgo LSP 수정**: TypeScript Go 포트(tsgo) 언어 서버가 정상 동작하도록 lsp-types 를 갱신하고, LSP 직렬화의 null / 빈 필드 처리 호환성도 함께 개선
- **Flex dock 너비 안정화**: 좌/우 dock 을 flexible 크기로 설정한 상태에서 센터 영역을 좌우로 split 하거나 활성 pane 을 이동해도 dock 비율이 흔들리지 않고 센터 열 수에 비례한 값으로 유지됨
- **Windows 앱 종료 시 크래시 방지**: Tokio 런타임 종료 경로에서 확장 작업이 panic을 일으키던 문제를 수정해 Dokkaebi를 닫을 때 비정상 종료가 발생하지 않음
- **Windows Git 상태 최신화**: 파일 변경 후에도 상태 패널과 변경 표시가 오래된 상태로 남던 문제를 수정해 Git 상태가 즉시 반영됨
- **Git 오류 토스트 크래시 방지**: 워크트리 생성 실패 등 Git 오류 알림이 뜨는 경로에서 발생하던 중복 대여 크래시를 수정
- **Git Graph 브랜치 변경 실시간 반영**: 브랜치를 새로 만들거나 지워도 HEAD 이동 없이 Git Graph 가 즉시 갱신됨
- **Git Graph stash 변경 실시간 반영**: `git stash push`/`pop` 이 Git Graph 에 바로 반영되어 stash 기록을 즉시 확인 가능
- **Markdown 프리뷰 밑줄·취소선 잔상 제거**: 프리뷰가 작은 크기로 축소돼 글자가 줄바꿈될 때 마지막 글자 뒤로 밑줄·취소선이 삐져나오던 문제 수정
- **단어 이동으로 접힘·@mention 건너뛰기**: Ctrl+좌/우 단어 이동이 접힌 코드 영역과 `@mention` 칩을 넘어가지 못하고 경계에 붙어버리던 문제 수정
- **테마 변경 시 브레드크럼 색상 즉시 갱신**: 테마를 전환해도 파일 경로 브레드크럼의 문법 색상이 이전 테마 값 그대로 남던 문제 수정
- **이름 변경한 파일 재시작 후 경로 유지**: 열려 있는 파일을 rename 하고 앱을 재시작하면 이전 경로로 열려 오류가 나던 문제 수정
- **활성 탭 마지막 탭 닫을 때 탭 전환기 유지**: 활성 pane의 마지막 탭을 닫을 때 탭 전환기(Ctrl+Tab)가 사라져 다른 탭으로 이동할 수 없던 문제 수정
- **정규식 검색 중 비매칭 강조 억제**: 정규식 검색 결과가 선택된 상태에서 regex 와 일치하지 않는 단어까지 occurrence 하이라이트가 겹쳐 표시되던 문제 수정
- **멀티버퍼 검색 결과 버퍼 semantic 하이라이팅**: 검색 결과에서 단일 파일을 열 때 semantic 토큰 하이라이팅이 초기 로드되지 않던 문제 수정
- **파일 복제 시 전체 이름 선택**: 파일을 복제해 rename 모드로 들어가면 `' copy'` 접미사만 선택되던 것을 파일명 전체 선택으로 변경해 새 이름을 바로 타이핑 가능
- **`-` 시작 경로 Git 명령 실패 방지**: `-`로 시작하는 파일명이 있는 저장소에서 blame·stage·unstage·stash·status가 옵션으로 오인되어 실패할 수 있던 문제 수정
- **ACP 서버 사용자 메시지 중복 방지**: opencode 등 일부 ACP 서버가 사용자 프롬프트 chunk를 에코해 동일 내용이 두 번 표시되던 문제 수정
- **에이전트 스레드 중복 세션 방지**: 동일 스레드를 연속으로 열면 같은 세션이 두 개 생성되던 문제 수정
- **Anthropic 커스텀 모델 thinking 모드 보존**: 사용자가 custom 모델에 설정한 thinking/adaptive-thinking 모드가 내장 모델 기준으로 덮어씌워지던 문제 수정
- **에이전트 도구 호출 포커스 유지**: 에이전트 패널을 스크롤해 도구 호출이 화면 밖으로 나가도 포커스 상태가 유지되도록 수정
- **제목 생성 실패 시 재시도 버튼**: 스레드 제목 자동 생성이 실패하면 타이틀 옆에 오류 아이콘이 표시되고 클릭해 재시도 가능
- **ACP 서버 종료 시 크래시 방지**: ACP 프로세스가 비정상 종료될 때 여러 세션을 순회하며 이벤트를 발생시키다가 중복 대여로 크래시 나던 문제 수정
- **에이전트 패널 최소 너비 보장**: 좌/우 도크에 고정된 에이전트 패널이 300px 미만으로 리사이즈되지 않도록 제한
- **큐 메시지 삭제 후 입력창 자동 포커스**: 대기 중인 메시지를 취소하면 다음 입력이 바로 가능하도록 메시지 입력창에 포커스 이동
- **Ollama 사용자 설정 컨텍스트 우선 적용**: 모델에 `num_ctx` 파라미터가 지정돼 있으면 모델 아키텍처 기본 컨텍스트보다 우선 사용
- **Git 패널 트리 Enter 키로 폴더 접기/펼치기**: 트리 뷰에서 디렉터리를 선택하고 Enter를 누르면 diff 대신 접기/펼치기 동작
- **Git diff hunk staging 동시성 수정**: Uncommitted diff가 Unstaged diff보다 먼저 열릴 때 staging 상태가 잘못 표시되던 경합 수정
- **Bare 저장소 경로 최근 프로젝트 보존**: `.bare` 디렉터리를 쓰는 워크트리 레이아웃에서 최근 프로젝트 경로가 사라지던 문제 수정
- **Git 명령 출력 ANSI 이스케이프 제거**: Git 실행 결과 창에 색 코드·진행 표시가 원시 `\x1b[...` 형태로 노출되던 문제 수정
- **TopoJSON 파일 문법 하이라이팅**: `.topojson` 확장자 파일이 JSON 문법으로 색상 표시되도록 자동 감지

### 새로운 기능 (Phase 3~5 백포트 추가분)
- **Ctrl+마우스 휠 에디터 폰트 크기 조정**: 에디터에서 Ctrl을 누른 채 휠을 굴려 폰트 크기 즉시 변경 가능
- **hover 팝업 고정·숨김 지연 설정**: `hover_popover_sticky`·`hover_popover_hiding_delay`로 hover 박스가 마우스 접근 시 유지되는 동작 제어
- **PNM(Netpbm) 이미지 프리뷰 지원**: `.pbm`·`.ppm`·`.pgm` 이미지 파일이 에디터에서 바로 열림
- **Fix with Assistant 코드 액션 제거**: 진단 컨텍스트에서 뜨던 "Fix with Assistant" 항목 제거, `assistant: inline assist` 액션은 그대로 유지
- **Format and Save 액션 추가**: `workspace: format and save` 액션으로 format_on_save 설정과 무관하게 항상 포맷 후 저장 가능
- **블록 주석 토글 단축키**: `Ctrl+K Ctrl+/` 또는 `Shift+Alt+A` 로 선택 영역을 `/* ... */` 같은 블록 주석으로 감싸거나 해제 (C/C++/Go/JS/JSONC/Markdown/Python/Rust/TSX 지원)
- **Git Graph 현재 브랜치 강조**: 커밋 옆 브랜치 칩 중 현재 체크아웃한 브랜치는 체크 아이콘 + 진한 배경으로 강조 표시. 긴 브랜치 이름은 자동 말줄임 처리로 폭 넘김 방지

### 새로운 기능
- **Windows 자동 실행 설정**: 설정 > 일반의 "자동 실행" 토글로 Windows 로그인 시 Dokkaebi 자동 실행을 켜고 끌 수 있음. 활성화하면 다음 로그인부터 자동으로 실행되고, 비활성화하거나 앱을 제거하면 자동 실행 등록이 해제됨
- **Claude Code 작업 알림**: Claude Code의 작업 완료·입력 대기·권한 요청을 Dokkaebi 토스트 알림으로 표시. 토스트 본문에 "그룹 / 탭" 라인이 함께 노출되어 발신 위치 식별 가능
- **터미널 탭/그룹 인디케이터**: 알림 도착 터미널 탭에는 dot, 해당 터미널이 속한 비활성 워크스페이스 그룹에는 배지 표시
- **알림 플러그인 관리**: 설정 > 알림 > Claude Code의 [설치]/[제거] 버튼으로 활성화 제어. 인스톨러에 번들되어 별도 다운로드 불필요

### UI/UX 개선
- **알림 메뉴 재구성**: "Claude Code 작업 완료 알림음" 항목이 "Claude Code 플러그인" + "작업 알림" 두 항목으로 재구성되어 플러그인 설치 상태가 화면에 즉시 표시됨
- **토스트 자동 사라짐 시간 조정**: 작업 완료·입력 대기 토스트는 기본 5초 후 자동으로 사라지고, 설정 > 알림 > "토스트 팝업" 섹션에서 5~300초 범위 조정 가능. 권한 요청 토스트는 사용자가 닫을 때까지 유지
- **토스트 중첩 방지**: 입력 대기 토스트가 떠 있을 때 새 작업 완료 알림이 도착하면 이전 토스트는 자동 정리. 여러 창에 같은 이름 탭이 있어도 발신 터미널이 속한 창에만 토스트 표시
- **Claude Code 토스트 내용 풍부화**: 작업 완료 시 직전 프롬프트와 응답 요약을 `"프롬프트 → 응답"` 형식으로, 권한 요청 시 도구 이름과 실행할 명령·파일 preview를, 입력 대기 시 Claude가 띄운 실제 질문을 토스트에서 바로 확인 가능
- **토스트 다국어 적용**: 토스트 제목·베이스 문구가 설정된 UI 언어(한국어/English)에 맞춰 즉시 전환
- **CLI 바이너리 아이콘 적용**: 설치 폴더와 탐색기에서 `dokkaebi-cli.exe`도 메인 앱과 동일한 도깨비 아이콘으로 표시되어 비주얼 일관성 향상
- **워크스페이스 그룹 패널 항목 확대**: 좌측 워크스페이스 그룹 패널의 항목 세로 높이를 약 30% 크게 조정해 클릭 영역과 가독성 개선
- **메모장 기본 복원 켜짐**: 설정 > 패널 > 메모장 복원 기본값이 켜짐으로 바뀌어 작성 중이던 메모가 앱 재시작 후에도 자동으로 유지됨

### 보안
- **rustls-webpki 보안 패치**: Dependabot 보안 경고 2건 해결을 위해 `rustls-webpki`를 0.103.11 → 0.103.12로 업데이트 (TLS 인증서 검증 관련 low 등급 경고 2건)

### 정리
- **마커 파일 기반 알림 시스템 폐기**: 기존 "Claude Code 작업 완료 알림음"이 사용하던 임시 파일 폴링 방식을 제거하고 IPC 기반 알림으로 전환. 기존 사용자 환경의 잔여 hook은 부팅 시 1회 자동으로 정리됨
- **언인스톨 시 Claude Code 훅 자동 정리**: 인스톨러로 Dokkaebi를 제거하면 `~/.claude/settings.json`에 등록된 Dokkaebi 플러그인 항목이 함께 제거됨. 사용자가 편집한 다른 Claude Code 설정은 보존되고, 자동 업데이트 재설치 경로에서는 hook을 유지해 업데이트 후 재등록 불필요

     ([자세한 내용은 공식 Zed 릴리즈 노트 참고](https://github.com/zed-industries/zed/releases/tag/v0.232.3))
