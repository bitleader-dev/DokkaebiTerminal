## 최근 변경
- 2026-04-14: Extension 마켓플레이스 로딩 실패 수정 — 라이선스 정리 2단계에서 `default.json`의 `server_url`을 `""`로 비웠던 부수 효과로 확장(Extensions) 탭이 "Failed to load extensions. Please check your connection and try again." 에러만 표시하던 문제 해결. **원인 분석**: `crates/extension_host/src/extension_host.rs:662`의 `fetch_extensions_from_api` 함수가 `http_client.build_zed_api_url(path, query)`로 URL을 만드는데, `crates/http_client/src/http_client.rs:214 build_zed_api_url` 함수 내부에서 base_url이 `"https://zed.dev"`일 때만 `"https://api.zed.dev"`로 변환하고 그 외 값(빈 문자열 포함)은 그대로 사용. `server_url=""` 상태에서는 `""+/extensions?query` → 유효하지 않은 URL 파싱 실패 → Extension API 호출 불가. **수정**: `build_zed_api_url`의 match 패턴에서 빈 문자열 케이스를 `"https://zed.dev"`와 동일하게 `"https://api.zed.dev"`로 fallback 매핑. `"https://zed.dev" | "" => "https://api.zed.dev"` 형태. Dokkaebi 기본 설치 상태(`server_url=""`)에서 Extension 마켓플레이스가 Zed Industries의 **public API**(익명 접근 가능)로 연결됨. 주석 추가: "Dokkaebi는 기본 server_url을 비워 Zed cloud 로그인을 차단하지만, Extension 마켓플레이스는 public API라 빈 값일 때도 api.zed.dev로 fallback한다." **비활성 유지 항목**: Zed cloud 자동 로그인은 `main.rs authenticate()` no-op으로 여전히 차단됨. UI의 Sign-In 버튼은 `sign_in_with_optional_connect`이 `server_url=""`로 네트워크 요청 생성 단계에서 실패(build_zed_url과는 별개 경로라 여전히 빈 URL). Extension API만 독립적으로 동작. **영향 평가**: (1) 사용자 혜택 — 테마·아이콘 테마·언어 서버 등 Zed 마켓플레이스 extension 설치·업데이트 가능. (2) 법적 리스크 — public API라 이용약관·상표법상 문제 없음(Zed Industries가 이 엔드포인트를 의도적으로 오픈 제공). (3) 의존성 리스크 — Zed Industries가 API 정책 변경/차단 시 영향받음, 자체 마켓플레이스 구축이 장기 대안. 빌드 검증: `cargo build -p Dokkaebi` 성공(3m 21s, 기존 unused import 경고 6건만, 본 작업 신규 경고·에러 없음).
